# Reference - Gérer opérationnellement les demandes entrantes liées aux droits des personnes
## RGPD pour organismes de formation

## 1. Objet du document

Ce document sert de base opérationnelle pour traiter les demandes d'exercice des droits des personnes dans un organisme de formation.

Il couvre notamment :
- le droit d'accès ;
- le droit de rectification ;
- le droit à l'effacement ;
- le droit d'opposition ;
- le droit à la limitation ;
- la gestion pratique des demandes manifestement infondées ou excessives ;
- l'articulation entre droits RGPD et obligations de conservation propres aux organismes de formation.

Public visé :
- direction ;
- DPO ;
- référent RGPD ;
- équipes administratives ;
- responsables qualité ;
- toute personne chargée de répondre aux demandes des stagiaires, formateurs, clients, prospects ou salariés.

---

## 2. Règles de base à appliquer

### 2.1 Principe général

Toute demande d'exercice de droits doit être :
- enregistrée ;
- datée ;
- qualifiée ;
- traitée dans un délai compatible avec l'article 12 du RGPD ;
- documentée ;
- clôturée avec une réponse formalisée.

### 2.2 Délai de réponse

Base légale :
- RGPD, art. 12.3

Règle :
- répondre dans les meilleurs délais ;
- au plus tard dans le mois suivant la réception de la demande.

Prolongation :
- possible de 2 mois supplémentaires si la demande est complexe ou si le nombre de demandes est élevé ;
- le demandeur doit être informé de la prolongation et de ses motifs dans le premier mois.

### 2.3 Gratuité

Base légale :
- RGPD, art. 12.5

Règle :
- l'exercice des droits est gratuit ;
- des frais raisonnables ne peuvent être demandés que pour des copies supplémentaires ou pour des demandes manifestement infondées ou excessives.

### 2.4 Vérification d'identité

Base légale :
- RGPD, art. 12.6

Règle :
- ne pas exiger systématiquement une pièce d'identité ;
- vérifier l'identité seulement en cas de doute raisonnable.

Exemples où l'identité peut être considérée comme suffisamment établie :
- demande envoyée depuis l'adresse e-mail habituellement utilisée par la personne ;
- demande effectuée depuis un espace authentifié ;
- demande cohérente avec des informations déjà connues.

Exemples où un doute raisonnable peut exister :
- adresse e-mail inconnue ;
- incohérences d'identité ;
- risque d'usurpation ;
- demande portant sur des données sensibles ou volumineuses.

Précaution :
- si une pièce d'identité est demandée, elle doit être transmise de manière sécurisée ;
- elle ne doit être conservée que le temps strictement nécessaire à la vérification.

---

## 3. Typologie des demandes à reconnaître

L'organisme doit être capable d'identifier rapidement la nature de la demande.

### 3.1 Droit d'accès
Base légale :
- RGPD, art. 15

La personne demande :
- confirmation que des données sont traitées ;
- copie des données ;
- informations sur les finalités, destinataires, durées, droits, source des données.

### 3.2 Droit de rectification
Base légale :
- RGPD, art. 16

La personne demande la correction d'une donnée inexacte ou incomplète.

### 3.3 Droit à l'effacement
Base légale :
- RGPD, art. 17

La personne demande la suppression de tout ou partie de ses données.

### 3.4 Droit d'opposition
Base légale :
- RGPD, art. 21

La personne s'oppose à certains traitements, notamment la prospection.

### 3.5 Droit à la limitation
Base légale :
- RGPD, art. 18

La personne demande la suspension temporaire de certains usages des données.

## 4. Procédure opérationnelle de traitement

## Étape 1 - Enregistrer la demande

Créer une trace interne avec au minimum :
- date de réception ;
- identité du demandeur ;
- canal de réception ;
- type de droit invoqué ;
- périmètre apparent de la demande ;
- date limite de réponse ;
- personne en charge ;
- statut du dossier.

## Étape 2 - Qualifier la demande

Vérifier :
- quel droit est exercé ;
- quelles catégories de données sont visées ;
- si la demande est suffisamment claire ;
- si une vérification d'identité complémentaire est nécessaire.

Si la demande est trop large ou imprécise :
- demander une précision sans bloquer artificiellement le traitement ;
- conserver la trace de la demande de précision.

Exemples de reformulation utile :
- "Souhaitez-vous uniquement votre dossier pédagogique ou également les données liées à votre espace en ligne ?"
- "Votre demande porte-t-elle sur vos données administratives, pédagogiques, comptables ou l'ensemble de votre dossier ?"

## Étape 3 - Vérifier les données concernées et les tiers

Point de vigilance :
- le droit d'accès ne doit pas porter atteinte aux droits et libertés d'autrui.

Base légale :
- RGPD, art. 15.4

Avant transmission :
- identifier les documents comportant des données de tiers ;
- masquer les noms, signatures ou éléments identifiants des autres personnes si nécessaire ;
- retirer les informations couvertes par le secret des affaires ou par un autre motif légitime de restriction, lorsque cela s'applique.

Exemple :
- une feuille d'émargement collective ne doit pas être transmise telle quelle si elle contient les noms et signatures des autres stagiaires.

## Étape 4 - Vérifier s'il existe une obligation légale de conservation

Cette étape est essentielle pour les demandes d'effacement.

La réponse ne doit pas être automatisée en "suppression totale" si certaines pièces doivent être conservées au titre :
- des contrôles administratifs ;
- des exigences des financeurs ;
- des obligations comptables ;
- de la défense en cas de contentieux ;
- des exigences qualité ou probatoires.

## Étape 5 - Répondre de manière motivée

La réponse doit être :
- claire ;
- loyale ;
- compréhensible ;
- complète ;
- traçable.

Elle doit préciser, selon le cas :
- ce qui est communiqué ;
- ce qui est rectifié ;
- ce qui est supprimé ;
- ce qui ne peut pas être supprimé ;
- pourquoi ;
- pendant combien de temps les données restantes seront conservées ;
- la possibilité d'introduire une réclamation auprès de la CNIL.

## Étape 6 - Archiver la gestion de la demande

Conserver une trace de :
- la demande ;
- la qualification ;
- les vérifications réalisées ;
- la réponse envoyée ;
- les documents transmis ;
- la date de clôture.

## 5. Demandes manifestement infondées ou excessives

Base légale :
- RGPD, art. 12.5

L'organisme peut :
- refuser d'y donner suite ;
- ou demander des frais raisonnables.

Cette décision doit rester exceptionnelle et être justifiée.

Exemples possibles :
- demandes répétitives à intervalles très rapprochés sans évolution de situation ;
- détournement manifeste du droit d'accès ;
- volume ou fréquence anormale sans motif légitime.

Bonne pratique :
- motiver précisément la décision ;
- rappeler le droit de saisir la CNIL.

---

## 6. Droit à l'effacement : limites spécifiques en organisme de formation

## 6.1 Principe

Le droit à l'effacement n'est pas absolu.

Base légale :
- RGPD, art. 17.1 et 17.3

L'effacement peut être refusé, en tout ou partie, lorsque le traitement reste nécessaire :
- au respect d'une obligation légale ;
- à la constatation, à l'exercice ou à la défense de droits en justice.

## 6.2 Cas fréquents d'impossibilité d'effacement total pour un OF

Peuvent devoir être conservés, selon le contexte :
- conventions ;
- contrats ;
- devis acceptés ;
- feuilles d'émargement ;
- certificats de réalisation ;
- attestations ;
- pièces justificatives liées au financement ;
- pièces comptables et factures ;
- éléments nécessaires à la défense en cas de contrôle ou contentieux.

## 6.3 Conséquence pratique

Lorsque l'effacement total n'est pas possible :
- supprimer ce qui peut l'être ;
- retirer les données des bases actives non nécessaires ;
- désinscrire des outils de prospection ou de communication ;
- basculer les données devant être conservées en archivage intermédiaire ;
- restreindre l'accès à ces seules données ;
- informer clairement la personne du refus partiel et de son fondement.

## 6.4 Archivage intermédiaire

Bonne pratique opérationnelle :
- isoler les données qui doivent encore être conservées ;
- restreindre l'accès aux seules personnes habilitées ;
- documenter la base légale, la durée et la finalité de conservation ;
- éviter que ces données restent dans les circuits d'usage courant.

---

## 7. Durées de conservation : logique à appliquer

Ce document ne fixe pas à lui seul toutes les durées applicables. La durée dépend :
- de la nature de la pièce ;
- de l'origine du financement ;
- des obligations comptables ;
- des exigences de contrôle ;
- du risque contentieux.

Règle de méthode :
- toujours distinguer base active et archivage intermédiaire ;
- toujours rattacher une durée à une base légale ou à une obligation identifiée.

Repères usuels à articuler avec les textes et financeurs applicables :
- pièces comptables : 10 ans à compter de la clôture de l'exercice, sous réserve des textes applicables ;
- pièces probatoires de réalisation : durée variable selon le régime de financement, le contrôle possible et les exigences des financeurs.

Point de vigilance :
- ne jamais annoncer une durée unique universelle pour tous les documents d'un OF ;
- vérifier les règles propres à l'OPCO, au CPF, à France Travail, aux fonds européens, au FSE, au PNRR ou à tout autre financeur concerné.

---

## 8. Cas particulier des données de santé

Point de vigilance majeur :
- un organisme de formation ne traite en principe des données de santé que de manière limitée et encadrée ;
- il faut distinguer les simples informations d'adaptation ou de handicap des dossiers médicaux proprement dits.

Règle :
- si la demande porte sur de véritables données médicales ou sur un régime juridique spécifique de communication, vérifier le cadre applicable avant de répondre ;
- ne pas appliquer automatiquement les règles générales sans cette vérification.

---

## 9. Anonymisation et BPF

L'anonymisation irréversible peut être une bonne pratique lorsque l'organisme a besoin de conserver des données d'activité sans conserver inutilement des données identifiantes.

Exemples d'usage :
- extraction statistique ;
- conservation de volumes d'heures ;
- suivi d'activité agrégé ;
- préparation d'éléments pour le BPF.

Point de vigilance :
- une pseudonymisation ne vaut pas anonymisation ;
- si la réidentification reste possible, le RGPD continue à s'appliquer.

---

## 10. Contenu minimal d'une réponse à un droit d'accès

La réponse à un droit d'accès devrait, selon le cas, inclure :
- confirmation du traitement ;
- copie des données personnelles concernées ;
- finalités du traitement ;
- catégories de données ;
- destinataires ou catégories de destinataires ;
- durée de conservation ou critères de détermination ;
- existence des droits complémentaires ;
- droit d'introduire une réclamation auprès de la CNIL ;
- source des données si elles n'ont pas été collectées directement ;
- information sur une éventuelle prise de décision automatisée si applicable.

---

## 11. Modèle de réponse - Droit d'accès

Objet : Votre demande d'accès à vos données personnelles

Bonjour [Prénom Nom],

Nous faisons suite à votre demande du [date] relative à l'exercice de votre droit d'accès.

Après vérification, nous vous confirmons que notre organisme traite des données vous concernant dans le cadre de [inscription / suivi pédagogique / facturation / gestion qualité / relation commerciale].

Vous trouverez ci-joint, par un canal sécurisé, une copie des données personnelles communicables vous concernant.

Les informations principales sur ce traitement sont les suivantes :
- Finalités : [à compléter]
- Catégories de données : [à compléter]
- Destinataires : [à compléter]
- Durée de conservation ou critères de conservation : [à compléter]
- Source des données, le cas échéant : [à compléter]

Certains éléments ont pu être masqués ou exclus lorsque leur communication portait atteinte aux droits de tiers ou relevait d'une restriction légalement justifiée.

Vous disposez également des droits de rectification, d'effacement, de limitation et d'opposition, dans les conditions prévues par le RGPD.

Si vous estimez, après nous avoir contactés, que vos droits ne sont pas respectés, vous pouvez adresser une réclamation à la CNIL.

Cordialement,

[Nom]
[Fonction]
[Coordonnées]

## 12. Modèle de réponse - Droit à l'effacement avec refus partiel

Objet : Votre demande d'effacement de données personnelles

Bonjour [Prénom Nom],

Nous faisons suite à votre demande du [date] relative à l'effacement de vos données personnelles.

Après analyse, nous avons procédé à la suppression ou à la désactivation des données qui n'avaient plus à être conservées, notamment :
- [exemple : adresse e-mail utilisée pour la prospection]
- [autres données supprimées]

En revanche, nous ne pouvons pas supprimer immédiatement l'ensemble de votre dossier.

En application de l'article 17.3 du RGPD, certaines données doivent être conservées lorsque leur traitement reste nécessaire au respect d'une obligation légale ou à la constatation, à l'exercice ou à la défense de droits en justice.

Sont notamment conservés, le cas échéant :
- [feuilles d'émargement]
- [certificats ou attestations]
- [conventions ou contrats]
- [factures et pièces comptables]
- [justificatifs liés au financement]

Ces éléments sont désormais conservés en archivage intermédiaire avec accès restreint, pour la durée strictement nécessaire au respect de nos obligations.

Si vous estimez que notre réponse n'est pas satisfaisante, vous pouvez introduire une réclamation auprès de la CNIL.

Cordialement,

[Nom]
[Fonction]
[Coordonnées]

## 13. Check-list opérationnelle interne

Avant de clôturer une demande, vérifier que :
- la date de réception a bien été enregistrée ;
- le délai de réponse a été calculé ;
- l'identité a été vérifiée de manière proportionnée ;
- la nature exacte du droit a été qualifiée ;
- les documents comportant des données de tiers ont été relus ;
- les obligations de conservation ont été analysées ;
- les suppressions réellement possibles ont été effectuées ;
- la réponse est motivée ;
- le droit de saisir la CNIL est rappelé ;
- l'ensemble du traitement de la demande est tracé.

---

## 14. Règles de prudence pour une IA ou une skill

La skill ne doit pas :
- conclure automatiquement à un effacement total ;
- supposer qu'une pièce d'identité est toujours exigible ;
- affirmer une durée de conservation sans vérifier le contexte ;
- ignorer les droits des tiers ;
- confondre anonymisation et pseudonymisation ;
- traiter les données de santé sans vigilance renforcée ;
- omettre la possibilité de refus partiel fondé sur l'article 17.3 du RGPD.

La skill doit :
- raisonner document par document ;
- distinguer suppression, désactivation, archivage intermédiaire et anonymisation ;
- rappeler les bases légales pertinentes ;
- signaler les points nécessitant une validation humaine ou juridique.

## 15. Bases juridiques minimales à citer

- RGPD, art. 12
- RGPD, art. 15
- RGPD, art. 16
- RGPD, art. 17
- RGPD, art. 18
- RGPD, art. 21
- Code de commerce, art. L123-22
- Référentiels et exigences des financeurs applicables selon le dossier
- Règles internes de conservation et politique d'archivage de l'organisme

---

## 16. Usage recommandé dans la skill

Ce document peut être mobilisé pour :
- analyser une demande d'exercice de droits ;
- générer une réponse conforme et proportionnée ;
- expliquer pourquoi certaines données peuvent ou non être effacées ;
- construire une procédure interne ;
- former une équipe administrative ;
- vérifier si un modèle de réponse est juridiquement prudent.

En cas de doute sur :
- la nature exacte des documents ;
- le financeur concerné ;
- la durée de conservation applicable ;
- la présence de données sensibles ;
- l'existence d'un contentieux ;
la skill doit recommander une validation humaine.
